# 总结生成开发日志

从项目根目录下的 conversations/ 文件夹下，将生成的文件按天进行总结，每天生成一份markdown总结，存入 .dev-logs/文件夹下。要求如下：
- conversations下的每个文件，启动一个subagent，来提取出来我的输入指令或选择、大模型提供的反馈（如选项、结果等，不要思考过程）；一般在 [step-finish part] 前后，前面是大模型的反馈，后面是我的选择或指令。
- 如果当天尚未结束，暂不总结。
- 如果.dev-logs/下已经有了之前某些日期的日志，不在重复生成。

subagent反馈内容：

### [session实现的功能总结]

本会话实现功能：功能一、

```bash
[时间戳]：[我的指令]

[时间戳]：[大模型提供的反馈]

[时间戳]：[我的选择]

[时间戳]：[大模型提供的反馈]

[时间戳]：[我的选择]

[时间戳]：[大模型提供的反馈]

...

```


最终格式模板如下：

文件名：[会话日期+xxxxx功能开发]

# [会话日期+xxxxx功能开发]

## 摘要

### 开发的功能清单

* 功能一
* 功能二

## 开发过程记录

### [session实现的功能总结]

本会话实现功能：功能一、

